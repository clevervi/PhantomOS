# Contributing to PhantomOS

First off, thank you for considering contributing to PhantomOS! It's people like you that make it a great tool for everyone.

## Code of Conduct
By participating in this project, you agree to abide by our Code of Conduct. (TL;DR: Be respectful and professional).

## How Can I Contribute?

### Reporting Bugs
- Use the **Bug Report** template.
- Describe the expected behavior vs the actual behavior.
- Include logs from `phantom_os.log`.

### Suggesting Enhancements/Tweaks
- We love new "Atomic Tweaks"!
- Use the **Feature Request** template.
- Explain the technical benefit of the tweak (Registry path, Value, etc.).

### Pull Requests
1. Fork the repo and create your branch from `develop`.
2. Ensure your code follows the existing style (XML comments are a must).
3. If you add a new tweak, ensure it's added to the `Catalog`.
4. Submit your PR against the `develop` branch.

## Coding Standards
- Use **XML Documentation** for all public methods and classes.
- Follow MVVM principles (Views in `Views`, Logic in `ViewModels`).
- Use `Logger` for any system-level interaction.
